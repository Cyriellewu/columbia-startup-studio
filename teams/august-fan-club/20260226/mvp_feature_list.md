# Evidence of Life — MVP Feature List

**Team:** August Fan Club  
**Date:** February 2026

These are the 3–5 core features required for a functional v1 of Evidence of Life. This is not a wishlist or long-term roadmap. The MVP focuses on the core transformation: scattered plans become visible proof of what actually happened.

---

## 1. Daily Plan + Timeline

Users can add tasks for the day and place them on a visual timeline.

**Core requirements:**
- Add, edit, and delete daily tasks
- Drag tasks into a timeline
- Assign planned start and end times
- Show tasks as timeline blocks
- Display planned time clearly so users can see what they meant to do

**Why it matters:** This turns a vague todo list into a concrete daily plan. For users with ADHD-style planning pain, the timeline makes time visible instead of keeping it abstract.

---

## 2. Focus Timer + Actual Time Tracking

Users can start a focus timer from a planned task, then save the actual work session.

**Core requirements:**
- Start, pause, resume, and stop a timer
- Connect timer sessions to specific tasks
- Save completed or partial focus sessions
- Show planned time vs. actual time
- Visualize gaps or deviation between the original plan and real activity

**Why it matters:** The timer is the bridge between intention and reality. A task is no longer only “done” or “not done”; even unfinished work can leave evidence.

---

## 3. Done List / Daily Recap

Users can review what actually happened during the day.

**Core requirements:**
- Show completed tasks and focus sessions
- Show unfinished but saved work sessions
- Allow users to carry unfinished work forward
- Display the day as a record of actions, not just a checklist
- Let users add short notes or recap entries

**Why it matters:** Many users end the day feeling like they did nothing because progress is invisible. The Done List gives users proof that they moved, even if the day did not go perfectly.

---

## 4. Moment Capture with Photos and Locations

Users can record lightweight moments with optional photos and location data.

**Core requirements:**
- Add a text moment
- Upload photos
- Attach optional location
- Show moments in daily recap
- Connect moments to calendar and map views

**Why it matters:** Not all evidence is a task. Photos, locations, and small notes help the product become a lightweight diary built naturally from what the user already did.

---

## 5. Calendar + Map Review

Users can look back across days through a calendar and map.

**Core requirements:**
- Calendar view showing tasks, moments, and daily records
- Map view showing location-based moments
- Basic filtering or navigation by day
- Link daily evidence back to recap and timeline

**Why it matters:** Evidence of Life should become more valuable over time. Calendar and Map turn daily records into a longer-term memory layer.

---

## Out of Scope for MVP

- Native mobile app
- Full social features
- Public sharing feed
- Complex AI memory chat
- Advanced analytics dashboards
- Full two-way Google Calendar editing
- Payments / premium subscription flow

These can be explored after the core loop is validated: plan → action → evidence → recap.
